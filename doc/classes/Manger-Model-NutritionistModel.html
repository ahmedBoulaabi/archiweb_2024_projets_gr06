addClientPlan()

NutritionistModel.php : 354

Adds a new plan for a client.

public addClientPlan(int $clientId, array<string|int, mixed> $recipesData, string $period, int $duration, string|null $plan_name) : bool

This method is responsible for adding a new plan for a client in the system. It first checks if the client already has an existing plan. If a plan exists, it deletes all existing plan-related entries for the client from the database. Then, it inserts a new plan into the 'plans' table and associates it with the client in the 'user_plan' table. It also adds the recipes included in the plan to the 'plan_recipes' table.

Parameters
$clientId : int

The ID of the client for whom the plan is being added.

$recipesData : array<string|int, mixed>

An array containing the details of the recipes included in the plan. Each entry should contain 'recipe_id' and 'date' keys.

$period : string

The period for which the plan is valid.

$duration : int

The duration of the plan in days.

$plan_name : string|null

The name of the plan. If not provided, a default name will be used.

Return values
bool —

Returns true if the plan is successfully added, false otherwise.

getUserProgressForNutritionist

public getUserProgressForNutritionist(mixed $nutritionistId) : array<string|int, mixed>|bool

Retrieves progress data for all clients associated with a specific nutritionist. Fetches detailed progress information for each client managed by the specified nutritionist, including client ID, full name, email, dietary goal, profile image, progress percentage, and plan creation date. Also calculates overall statistics such as total users, users who have not completed their plan, and users who have completed their plan.

Parameters
$nutritionistId : mixed
Return values
array<string|int, mixed>|bool —

JSON-encoded array with success message and data (total_users, not_completed, completed, users_progress) if progress data is found. JSON-encoded array with error message if no data or nutritionist ID not provided.
